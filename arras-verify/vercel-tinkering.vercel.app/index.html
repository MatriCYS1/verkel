Chat: Rules

Note: these rules are just guidelines and we reserve the right to take action if we believe the content you post violates community standards. The rules include but are not limited to:

  • No NSFW Content: Images and media containing extreme blood, gore, violence and/or adult content are strictly prohibited and will result in an upload ban.
  • No Harmful Speech: Offensive text such as racial slurs and/or verbal harassment will result in a mute or a ban.
  • No Spam: Sending many messages or pings in short periods of time and/or large text walls will result in a mute.
  • No Advertising: Promoting irrelevant links and/or other media will result in a mute.
  • No Exploiting: Circumventing punishments will result in more severe consequences, such as account suspension.

Public Commands

/rlq
get your account's rate limit info
/dm [user id] [message]
send a private message to another user
/roll [side count]
roll a dice with the given amount of sides
/emotelist
get the list of every available emote
/servers [list of modifiers]
get data for the current arras.io servers, more detailed information is available in the changelog (v1.14 and v1.14.3)

Other Commands

/cc
clear the chat (requires Director rank or higher)
/clear
alias for /cc (requires Director rank or higher)
/mute {{[username]}} [duration]
mute a user; more examples are available below (requires Director rank or higher)
/mute {{Testing}} 2h
mute user "Testing" for 2 hours (requires Director rank or higher)
/mute {{upload:Testing}} 2d
ban user "Testing" from uploading media for 48 hours (requires Director rank or higher)
/muteguests
mutes/unmutes all guests (requires Director rank or higher)
/restart
refresh every user's client (requires Trial rank or higher)

29th April 2026 - v1.14.3

  • Added more features to /servers. The newly added features are:
    • The sorting modifier can now use aliases.
      • "clients" can be aliased as "c", "p", "cli" or "players".
      • "uptime" can be aliased as "u", "t" or "time".
      • "mspt" can be aliased as "m" or "ms".
    • Sorting can be reversed by using two $ signs. For example, $c sorts by highest clients and $$c by lowest clients.
    • Inequalities can be created with the ~ sign. For example, ~clients<50 filters by servers with less than 50 players.
      • Multiple inequalities can be used in one /servers call, and aliases can be used for the left hand side.
      • The available operators are "<", ">", "=", "<=" and ">=".
      • When creating an inequality with uptime, the numerical part is measured in seconds. For example, "~u<3600" filters by servers with less than 1 hour of uptime.

23rd April 2026 - v1.14

  • Added /g local, allowing any user to play /g on their client only.
  • Added 4 new toggleable settings.
  • Readded /servers, using a new syntax system.
    The patterns able to be used are:
    • "x" - show servers with at least 1 player only
    • "s" - show sandboxes only
    • "m" - show non-sandboxes only
    • "(value1)^(value2)" or "(value1)~(value2)" - show servers in a given range, value1 and value2 can be integers, "x" or nothing. For example, "^5" gets the top 5 servers, "~10" the top 10, "5^10" the servers from 5th to 10th" and "~" gets every server result.
    • "#wco" - filter by certain regions (for example #wco = us west, us central and oceania)
    • "#!wpd" - filter by one server (for example #!wpd = show #wpd only)
    • "$uptime" or "$mspt" - change sorting mode, defaults to $clients
